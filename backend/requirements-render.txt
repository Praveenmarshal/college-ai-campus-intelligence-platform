# ============================================================
# requirements-render.txt
# Slimmed-down dependency set for Render's FREE TIER deployment.
#
# Render free tier limits: 512MB RAM, shared CPU, slow/limited build
# minutes. The full requirements.txt pulls in PyTorch (via easyocr +
# sentence-transformers), which alone is 2-3GB and will fail to build
# or OOM-crash on the free instance type.
#
# What's DIFFERENT from requirements.txt:
#   - sentence-transformers / embeddings → still included (needed for
#     PDF RAG chunk embedding), but pinned to CPU-only torch explicitly
#   - easyocr REMOVED → OCR endpoints (/api/ocr/*) will respond with a
#     clear "not available on this deployment" error instead of crashing
#     the whole app. pytesseract is kept as a much lighter OCR fallback.
#   - xgboost REMOVED → ML prediction falls back to the built-in
#     scikit-learn models + heuristics (Predictor already has this
#     fallback path built in — see services/ml/predictor.py)
#   - celery/redis REMOVED → not needed for a single-instance demo
#
# Use this file in Render's "Build Command":
#   pip install -r requirements-render.txt
#
# For full local/Docker/production deployment with every feature
# enabled, use the standard requirements.txt instead.
# ============================================================
